记录一次博客的搭建过程

原因：

和大多数人的原因一样，对于博客这种轻量级的系统，我是懒的自己去搞服务器做动态的系统了。刚好GitHub pages提供静态页面解析，我之前一直在使用hexo博客系统，然而最近它的编译速度实在是慢，而且命令格式有点奇怪，刚好了解到Hugo这个使用go语言编写的博客系统编译速度非常快，遂动手使用Hugo搭建博客系统了。

一、准备工作

电脑先安装Hugo编译系统，不过hugo是使用Go编写的，所以可能需要先安装Go环境（也可能安装hugo的时候会自动安装go环境），不过我电脑之前安装好了Go环境，所以我直接安装hugo就行了。直接找到hugo官网，打开它的Docs，按照它的教程来即可。例如我使用的是MacOS系统，按照Hugo命令如下(如果没有安装brew的话，先自行安装)：

1

$brew install hugo

测试一下是否安装成功：

1

$hugo version

有输出就证明安装成功了。

在本地家目录下建一个网站

1

$hugo new site MyBlog

网站名字自己随便起。

进入该站点目录，随后把该目录使用git管理，因为以后需要使用git把本地的修改推送到github上去。

1

$git init

二、选择一个博客主题

可以到hugo官网的博客站点themes.gohugo.io选择一个你喜欢的主题，我选择的主题是even,之后便可以按照该主题的README来写了到你的本地博客站点目录下运行一下命令把该主题克隆下来：

1

$ git clone https://github.com/olOwOlo/hugo-theme-even themes/even

把它的主题文件下的exampleSite 目录下有一个 config.toml 文件，将这个 config.toml 文件复制到你的站点目录下，根据自己的需求更改即可。在博客站点根目录下运行：

1

$ cp themes/even/exampleSite/config.toml ./

之后用文本编辑器vim打开该配置文件，根据自己的需要修改保存即可。

第一篇博客

在even主题的readme中我们知道，对于这个主题，使用的是post而不是posts，所以我们使用如下命令：

1

$ hugo new post/my-first.md

这会在content/post目录下生成一篇md文件 打开该markdown文件我们可以观察到该文件头部的元数据区记录了该文件的生成时间和draft：true的选项，这是通过模版生成的，后面可以修改。 现在我们来运行一下hugo自带的web服务器来测试一下我们的博客站点有没有搭建成功：

1

$ hugo server -D

-D选项表示我们可以把该draft为true的博客也运行显示，即草稿也显示出来，不过后面我们需要编译生成静态站点页面时我们需要修改draft为false。现在打开http://localhost:1313/应该就可以看到自己的博客页面了。

三、把本地博客推送并部署到GitHub上去

当然，Hugo也贴心地给我们写好了部署教程，具体可以参考Host on GitHub。Github pages分为两种：

  • User/Organization Pages (https://.github.io/)
  • Project Pages (https://.github.io/)

即一个是个人或组织的页面，一个是项目页面。在这里我选择的是把博客网站部署到我的github个人页面上去。hugo官网建议我们在Github上建立两个库，一个是保存你的博客站点的所有源文件，一个库用来渲染发布博客站点（即编译后生成的静态文件）。

建立远程库

  1. 在githu上创建一个名字为blog（任意取）的库来保存我们的博客站点源文件，该库可以是公有库或私有库（2019年，Github宣布无限免费私有库，有个有钱的微软爹真好），我个人是把该库创建为私有的库，因为后期可能需要上传一些私人的东西。 2.创建一个<username>.github.io的公有库，该库即用来显示你的博客静态页面的，其中的username换成自己的github名字。

我把远程库和本地站点库对应了

1

git remote add origin git@github.com:username/blog.git

注：如果你是直接建立的远程库并clone下载再建立网站的，可以不需要我这么麻烦。 现在让我们编译一下站点，运行：

1

$ hugo

我们生成了public文件夹，该文件夹实际上就是我们用来推送到我们的远程库<username>.github.io的文件。 所以我们需要把该文件夹与该远程库想关联，我们先运行rm -rf public（注意此命令一定要在博客网站根目录下运行且-rf后面一定不能是/ 否则你整个电脑的东西都没有了）删除刚才编译生成的public文件夹。

添加子模块

1

$ git submodule add -b master git@github.com:<USERNAME>/<USERNAME>.github.io.git public

上面的username换成你自己的github名字。 这样我们就把该public与<username>.github.io远程库建立了联系。这样推送的时候我们就可以分别把整个站点源文件推送到blog(刚才建立的远程库)和public网站静态文件夹推送到<username>.github.io远程库中。

自动化部署脚本

之前，我试过使用travis ci来自动化部署的，但是有个很重要的原因让我放弃使用：

  • 自动化部署拿来编译网站时，每次build时都会重新下载编译依赖环境，这样速度就慢了，至少几分钟（之前就是因为hexo编译慢，再加上这个不就更慢了吗。），而hugo本地编译不到1s中加上推送到github上的时间也不过几秒中。所以基本上能立刻看见效果。

所以我打算在本地编译推送，然而我们自己每次都要hugo,git add和git commit再git push的重复动作，繁琐。所以想到写一个简单的shell脚本，没想到的是hugo官网这都替我们想好了自动化部署脚本。直接拿来改一下就行了。 由于我的本地库环境与官网说的有点不一样（上面提到），我的需要push两次远程库。

#!/bin/bash

echo -e "\033[0;32mDeploying updates to GitHub...\033[0m"

# Build the project.
hugo # if using a theme, replace with `hugo -t <YOURTHEME>`

# Go To Public folder
cd public
# Add changes to git.
git add .

# Commit changes.
msg="rebuilding site `date`"
if [ $# -eq 1 ]
  then msg="$1"
fi
git commit -m "$msg"

# Push source and build repos.
git push origin master

# Come Back up to the Project Root
cd ..

# add my personal push

git add .

git commit -m "update my blog"

git push origin master

上述脚本最后的add my personal push是我自己添加的，因为要push两次。给上述脚本添加执行权限

1

$ chmod +x deploy.sh

最后我们新写了一篇博客或者修改后（记得把博文的draft改成false，或者直接修改archetypes/default.md里的draft为false，这样每次都不用修改了），我们就可以自己运行

1

$ ./deploy.sh

就会推送到github上去了，非常方面，最后我们打开我们的个人github页面网址<username>.github.io就可以看到我们的博客了。

四、添加域名

如果你想要自己的域名好看有好记的话，可以去购买域名，域名购买国内可以直接阿里云购买即可，购买后在域名管理后台添加该域名解析到你的github个人页面网址上即可。

添加了域名解析后，我们需要在我们的page库里增加我们自己域名即创建一个CNAME的文件，里面的就输入一个我们的域名即可。由于我们每次编译时都会自动覆盖之前的public，我们不能直接在public下添加，我们在在我们本地博客站点的static目录下添加该文件，之后输入域名比如：jack.com啥的即可，不需要www。 之后我们每次编译生成的public文件就会自己带有CNAME了。 最后我们就可以通过自己的添加的域名访问了。

五、添加评论系统valine

为什么使用valine而不使用gittalk或者其它的评论系统呢？是因为此评论系统不需要登陆而能任意评论。我个人在别人博客下评论时看到需要登陆才能评论的限制，我其实是挺反感的。所以自己的这个博客系统没有添加这个限制。 如何使用valine？请参考valine官网教程。

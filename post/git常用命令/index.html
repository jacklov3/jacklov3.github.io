#进入一个目录，会初始化一个空的git仓库，会生成一个.git目录
$git init
#运行 git status如果你的目录中有文件，可以看到命令输出提示所有未跟踪的文件
$git status
#但是有些文件我们不想跟踪并提交到版本库中，我们可以在目录下建立一个".gitignore"文件，里面填入不想被跟踪的文件，如程序生成的一些字节码，缓存，mac下的.DS_store .vscode的文件夹，你可以自己写也可以直接使用https://github.com/github/gitignore 里面的模版，不过添加一些自己不希望push的东西。

#添加文件到暂存区，或者直接git add . 添加所有文件
$git add 文件名

#提交快照，git中被称为commit。一旦你把文件改乱了，或者误删了文件，还可以从最近的一个commit恢复，然后继续工作，而不是把以前的工作成果全部丢失，这就是版本控制的好处

$git commit -m "提交的说明，方便以后查看"

#使用git log 或简洁版的git log --pretty=oneline可以查看提交历史
$git log
或
$git log --pretty=oneline

# 回退到某个历史版本，HEAD指向的版本就是当前版本，因此，Git允许我们在版本的历史之间穿梭，使用命令git reset --hard commit_id，其中commit_id可以由上一步查看得到，输入前几个字符按table补全即可。如果想回到上几个版本，其中commit_id可以依次为HEAD^，HEAR^^...
$git reset --hard commit_id

#注意:若你回退到以前的版本，你会发现git log输出的提交中，未来的版本commit已经没有了，这个时候你就要使用git reflog查看命令历史，以便确定要回到未来的哪个版本
$git reflog

与github远程库关联

在本地创建git仓库，肯定要推送到远端备份了，全球最大的代码托管平台github肯定是你不二的选择，不过如果有时候遇见网络问题那你就得自己想办法了😄。好消息，2019年github已经宣布个人可以免费建立无限个私有库了。

  • 我们先在github上建立一个库，建立时最好不要选择生成README.md，因为这会使的你的远程仓库已经有了一次commit了，本地已经有的git库push上去需要解决commit历史不同的问题。

  • git remote add origin git@github.com:/.git添加远程库到本地git库中

  • 如上面提到的问题，你可以选择git clone 远程库下来再来关联，若使用本地已经有的git库关联远程库，则若远程库中有commit历史，则要解决commit历史不太的问题（使用git pull origin master –allow-unrelated-histories可以让不相关commit历史合并）

  • 接下来第一次使用git push -u origin master可以推送到远程库并建立关联以后推送或拉取就可以直接简写成git push即可。

哦对了，第一次使用git clone或git push可能会产生ssh密钥的问题，你需要同意，并把本地的ssh公钥添加到自己的github上，这样才能推送。本地的ssh目录一般放置在用户目录下。 暂时就这么多吧，我暂时还没有使用分支的情况，自己太菜，还没和别人合作。😢

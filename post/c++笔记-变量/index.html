C++笔记–变量与基本数据类型

基本类型

  • 算术类型：整型（整型数、字符、布尔类型）、浮点型
  • 空类型(void)

除去布尔型和扩展的字符型外，其它整型可以划分为带符号的（signed)和无符号的（unsigned)两种。与其它整型不同的是，字符型被分为了三种：char、signed char和unsigned char。

类型转换

c++中的内置类型一般都能进行自动类型转换，但是要注意几点：

  • 非布尔类型的值赋值给布尔型时，初始为0时结果为false，否则结果为true。
  • 把布尔赋值给非布尔时，初始值为false则结果为0，初始值为true则结果为1。
  • 把浮点数赋予给整型时，结果只保留浮点数中小数点之前的部分，即丢失精度。
  • 整数赋给浮点类型时，小数部分记为0。若该整数超过了浮点类型的容量，可能丢失精度。
  • 当赋给无符号类型一个超出它表示范围的值时，结果是初始值对无符号类型表示数值总数取模后的余数。（如8比特的unsigned char表示0-255的值，即对256取模），把-1赋给它则结果就是255。
  • 赋值给带符号类型一个可能超过表示范围的值时，结果是未定义的。

变量

变量声明和定义，变量和对象很类似，只不过变量时c++内置对象的说法，而对象是用户自定义的类的实例的说法。

初始化

首先初始化和赋值是不同的操作。初始化是指对象调用构造函数初始化对象的值，而赋值是指把对象的当前值擦除，而以一个新值来替代。

至于初始化的形式有几种：

1
2
3
4

int i = 0;//这种初始化形式看起来与赋值一样，实则是操作符重载，在对应的类中定义了相应的=号操作，可能调用了对应类型的构造函数
int i = {0};
int i(0);//看起来就是调用构造函数创建一个对象
int i{0};//列表初始化(c++11标准)

默认初始化

内置类型的变量如果定义在所有块之外(即全局变量),则被初始化为0，否则如果是块内的内置类型则不被初始化，此时的值是未定义的，使用未定义值的变量可能引发程序错误。

  • 每个类各自决定其初始化对象的方式，例如string类规定如果没有指定初值则生成一个空串。

变量声明和定义

C++ 支持分离式编译，允许将程序分割成若干个文件，每个文件可被独立编译。为支持独立编译，C++语言将声明和定义分开来。声明使得对象为程序所知，一个文件如果想使用别处定义的名字则必须包含对那个名字的声明。而定义则负责创建与名字关联的实体。

如果想声明一个变量而非定义它，则在变量前加extern，而且不要显式地初始化变量。

1
2
3

extern int i;//声明i而非定义i
int j;//声明并定义j
extern int i =2;//这属于定义了，消除了extern的作用

变量能且只能被定义一次，但是可以被多次声明。

如果要在多个文件中使用同一个变量，就必须将声明和定义分离。此时，变量的定义必须出现在且只能出现在一个文件中，而其他用到该变量的文件必须对其进行声明，却绝对不能重复定义。

具体使用场景例如：包含在头文件<c.h>中声明一个变量

1

extern int h;

在<c.pp>中定义了该变量

1

int h = 3;

则在其他.c++文件中include这个<c.h>文件就可以直接使用h=3的变量了。

include相当于在源文件中加入该段代码。

变量作用域

名字main和其他定义在函数体外的名字一样有全局作用域。作用域能彼此包含，内层作用域可以访问外层作用域中的变量，就像在main函数中可以访问全局变量一样。内层作用域的作用范围就式块作用范围。如函数内，或者for循环内。内层作用域中的变量允许和外层作用域一样，但是在块内优先访问局部变量，要访问外层相同名字的变量则要使用作用域::

复合类型

符合类型是基于其它类型定义的类型，如引用、指针等。

引用

引用即为对象起的一个别名，使用&符号来定义引用类型。 引用把自己和引用的对象绑定在一起，而且必须初始化，一旦初始化完成，引用将和它的初始对象一直绑定在一起。因为无法令引用重新绑定到另外一个对象，因为引用必须初始化。

引用分左值引用，右值引用（c++11新增），一般我们讲的是左值引用。 引用就是对象的别名，就像给一个箱子贴上另一标签，所以对引用的操作都会作用到原对象上。引用不是对象。

1
2

int x = 2;
int &a = i;//引用必须初始化，即确立绑定关系，之后无法解除

指针

指针是一复合类型，与引用类似,指针也实现了对其他对象的间接访问。然而指针与引用相比有不同点：

  • 指针本身是一个对象，允许对指针赋值和拷贝，而且在指针的生命周期内它可以先后指向几个不同的对象。
  • 指针无须在定义时赋初值。和其他内置类型一样，在块作用域内定义的指针如果没有被初始化，也将拥有一个不确定的值。
1
2
3

int i =2;
int *p=&i;//定义了一个指针变量p指向整型i对象，&用于获取变量的地址。
*p = 3;//相当于i=3，通过指针间接改变指向的对象的值。*指解引用操作。

int型，double型等什么指针指的是该指针变量指向的对象的类型，而非指针变量本身的类型，指针变量存储的是地址(一个16进制的值)。

空指针不指向任何对象，在试图使用一个指针之前代码可以先检查它是否为空

int *p1 = nullptr;//等价于int *p1=0;c++11标准
int *p2 =0; //直接将p2初始化为字面常量0
int *p3=NULL;//等价于 int *p3 = 0;

//判断指针是否为空，只需将指针变量用于if判断语句中
if(p)

void * 指针是一种特殊的指针类型，可用于存放任意对象的地址。一个void*指针存放着一个地址。不能直接操作void* 指针所指的对象，因为我们并不知道这个对象到底是什么类型，也就无法确定能在该对象上做哪些操作。

指向指针的指针

1
2
3

int i = 1024;
int *p = &i;//p指向一个int型的数
int **pp = &p;//pp指向一个int型的指针

指向指针的引用

引用本身不是一个对象，因此不能定义指向引用的指针。但指针是对象，所以存在指针的引用:

1
2
3
4
5

int i = 3;
int *p;//p是一个int型指针
int *&r = p;//r是一个对指针p的引用，也就是指针变量p的别名
r = &i;//r引用了一个指针，因此给r赋值&i就是令p指向i
*r=0;//解引用r得到i，也就是p指向的对象，将i的值改为0

对一条比较复杂的指针或引用的声明语句，从右向左阅读比较好。

const限定符

const限定一个变量为常量，则此变量的值不能被改变

1
2

const int bufsize =512;
bufsize=1024//❌，不能改变const对象的值。

注意：在不改变const对象下，用一个对象去初始化另一个const对象。则它们是不是const都无关紧要

1
2
3

int i = 42;
const int ci = i;//正确，i的值被拷贝给整型常量ci，即使i是非常量
int j =ci;//即使ci是常量，但是该赋值只是拷贝ci的值给j，之后j的值怎么操作与ci无关，j的值可以改变。

默认情况下，const对象被设定为仅在文件内有效。当多个文件中出现了同名的const变量时，其实等同于在不同文件中分别定义了独立的变量。

如果要在文件间共享const变量，则应该在const变量不管时声明还是定义都添加extern关键字，这样只需定义一次就行了：

1
2
3
4

//file_1.cc定义并初始化了一个常量，该常量能被其他文件访问
extern const int bufsize = fcn();
//file_1.h头文件
extern const int bufsize;//与file_1.cc中定义的bufsize是同一个。

const的引用可以把引用绑定到const对象上，就像绑定到其他对象一样，我们称之为对常量的引用。与普通引用不同的是，对常量的引用不能被用作修改它所绑定的对象：

1
2
3
4

const int ci = 1024;
const int &r1=ci;//正确：引用及其对应的对象都是常量
r1 = 42;//错误：r1是对常量的引用
int &r2 = ci;//错误，试图让一个非常量引用指向一个常量对象

初始化和对const的引用

引用的类型必须与其所引用对象的类型一致，但是有两个例外：

  • 在初始化常量引用时允许用任意表达式作为初始值，只要该表达式能转换成引用的类型即可。
1
2
3
4
5
6
7
8

  int i =42;
  const int &r1 = i;//允许将const int&绑定到一个普通int对象上
  const int &r2= 42;//正确：r2是一个常量引用
  const int &r3= r1*2;//正确：r3时一个常量引用
  int &r4 = r1 *2;//错误:r4是一个普通的非常量引用
  double d = 3.14;
  const int &ri = d;//ri是一个整型的引用，我们把一个浮点型赋予它，则会创建一个临时的整型变量（临时量），所以我们的ri实际绑定的是这个临时量。
  int &ri = d;//若ri是指向非常量的引用，把浮点数赋值给它则是非法的。

对const的引用可能引用一个并非const的对象

常量引用仅对引用可参与的操作作出了限定，对于引用的对象本身是不是一个常量未作限定。因为对象也可能是个非常量，所以允许通过其他途径改变它的值：

1
2
3
4
5

int i =42;
int &r1=i;//引用r1绑定对象i
const int &r2 =i;//r2也绑定对象i,但是不允许通过r2修改i的值
r1 =0;//r1并非常量，i的值修改为0
r2 = 0;//错误：r2是一个常量引用

指针与const

与引用一样，也可以令指针指向常量或非常量。类似于常量引用，指向常量的指针不能用于改变其所指对象的值。要想存放常量对象的地址，只能使用指向常量的指针：

1
2
3
4

const double pi = 3.14;//pi是一个常量，它的值不能改变
double *ptr = &pi;//错误：ptr是一个普通指针
const double *cptr =&pi;//正确，cptr可以指向一个双精度常量
*cptr = 42；//错误：不能给*cptr赋值

指针的类型必须与其所指对象的类型一致，但是有两个例外：

  • 允许令一个指向常量的指针指向一个非常量对象：
1
2

  double v = 3.14//v是一个双精度浮点数，它的值可以改变
  cptr = &v；//正确：但是不能通过cptr改变v的值

和常量引用一样，指向常量的指针也没有规定其所指的对象必须是一个常量。所谓指向常量的指针仅仅要求不能通过该指针改变对象的值，而没有规定那个对象的值不能通过其他途径改变。

const指针

指针是对象而引用不是，因此就像其他对象类型一样，允许把指针本身定为常量。常量指针必须初始化，而且一旦初始化完成，则它的值（也就是存放在指针中的那个地址）就不能改变了。把* 放在const关键字前用以说明指针是一个常量。

1
2
3
4
5
6

int i=3;
int *const p = &i;//p将一直指向i
*p=34;//可以通过常量指针p改变i的值，因为指针是常量，但是所指向的对象是非常量（可以改变的）
const double pi = 3.14159;
const double *const pip = &pi;//pip是一个指向常量对象的常量指针
*pip=2.3；//错误。pip是一个常量指针，它指向的对象也是一个常量不能被改变

顶层const和底层const

指针本身是一个对象，它又可以指向另外一个对象。因此，指针本身是不是常量以及指针所指的是不是一个常量就是两个相互独立的问题。用名词顶层const(top-level const)表示指针本身是个常量，而用名词底层const(low-level const)表示指针所指的对象是一个常量。

更一般的,顶层const可以表示任意的对象是常量，这一点对任何数据类型都适用。底层const则与指针和引用等复合类型相关（指指向的对象是常量）。特殊的是指针类型即可以是顶层const也可以是底层const

1
2
3
4
5
6

int i = 0;
int *const p1 = &i;//顶层const，不能改变p1本身的值
const int ci =42;//不能改变ci的值，是一个顶层const
const int *p2 = &ci;//允许修改p2的值但指向的是常量，底层const
const int *const p3 = p2;//靠右的const是顶层constant，靠左的是底层const表示p3是一个常量指针，该指针指向的对象又是个常量，p2是个指向常量的指针，存储了该常量的地址，把p2赋给p3，指两个指针指向相同的常量对象的地址
const int &r=ci;//用于引用的const都是底层const（指引用的对象是常量），不存在引用是常量的，因为引用本身不是一个对象

constexpr和常量表达式

常量表达式是指值不会改变并且在编译过程就能得到计算结果的表达式。 c++11新增加了constexpr类型来让编译器来验证变量的值是否是一个常量表达式。

1
2
3

constexpr int mf = 20; //20是常量表达式
constexpr int limit = mf+1;//mf+1是常量表达式
constexpr int sz = size()//只有size是一个constexpr函数时才是一条正确的声明语句

指针和constexpr

在constexpr声明中如果定义了一个指针，限定符constexpr仅对指针有效，与指针所指的对象无关：

1
2
3

const int *p = nullptr;//p是一个指向整型常量的指针
constexpr int *q = nullptr;//q是一个指向整数的常量指针，constexpr定义为顶层const
constexpr int *p1 = &j;//p1是常量指针，指向整数j

decltype

它的作用是选择并返回操作数的数据类型，

1
2
3
4

const int ci= 0,&cj=ci;
decltype(ci) x =0;//x的类型是const int 
decltype(cj) y =x;//y的类型是const int&,y绑定到变量x

需要指出的是，引用从来都作为其所指对象的同义词出现，只有用在decltype处是一个例外。

1
2
3
4

int i=3;*p=&i,&r=i;

迭代器

首先，迭代器不是指针，它只是类似指针的操作，是一个模版类，重载了指针的操作，它用于访问容器内的元素，返回的是指向元素的迭代器，需要解引用

1
2

*iter //返回迭代器iter所指元素的引用
iter->mem//解引用iter并获取该元素的名为mem的成员，等价于(*iter).mem，是它的简写

限制：不能在范围for循环中向vector对象添加元素，另一个限制是任何一种可能改变vector对象容量的操作，比如push_back，都会使该vector对象的迭代器失效。谨记，但凡是使用了迭代器的循环体，都不要向迭代器所属的容器添加元素。迭代器之间没有加法，只有减法。

###左值和右值

对左值和右值的一个最常见的误解是：等号左边的就是左值，等号右边的就是右值。左值和右值都是针对表达式而言的，左值是指表达式结束后依然存在的持久对象，右值是指表达式结束时就不再存在的临时对象。一个区分左值与右值的便捷方法是：看能不能对表达式取地址，如果能，则为左值，否则为右值。

​ 区分清楚了左值与右值，我们再来看看左值引用。左值引用根据其修饰符的不同，可以分为非常量左值引用和常量左值引用。非常量左值引用只能绑定到非常量左值，不能绑定到常量左值、非常量右值和常量右值。如果允许绑定到常量左值和常量右值，则非常量左值引用可以用于修改常量左值和常量右值，这明显违反了其常量的含义。如果允许绑定到非常量右值，则会导致非常危险的情况出现，因为非常量右值是一个临时对象，非常量左值引用可能会使用一个已经被销毁了的临时对象。

常量左值引用可以绑定到所有类型的值，包括非常量左值、常量左值、非常量右值和常量右值。

  • 非const左值引用只能绑定到非const左值；
  • const左值引用可绑定到const左值、非const左值、const右值、非const右值；
  • 非const右值引用只能绑定到非const右值；
  • const右值引用可绑定到const右值和非const右值。

小结

引用不是对象，若用const修饰引用，则表示该不能用该引用改变引用的对象。

常量指针和指针常量：

常量指针指的是指针是常量，其对应的地址值不能改变，永远指向一个对象，必须初始化，但是所指向的对象是常量还是非常量与指针无关，可以通过该指针改变所指对象的值。

指针常量（指向常量的指针），指指针所指向的对象是个常量不能改变。

本身是常量的必须初始化。

重要:无论是指针还是迭代器，如果所指对象是const，则必须声明指针或者迭代器为const类型，但是如果所指对象为为非const，则指针和迭代器可以为非const，也可以为const，具体情况具体分析。
